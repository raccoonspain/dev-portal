# Дизайн: Дашборд ресурсов сервера

**Дата:** 2026-06-28  
**Статус:** Согласован  
**Контекст:** Dev Portal — персональный веб-портал на VPS (Node.js + Express, ванильный JS/HTML/CSS)

---

## Цель

Добавить вкладку «Сервер» на страницу `dev-portal`, показывающую текущую загрузку CPU, RAM, диска и таблицу всех процессов с возможностью сортировки.

---

## Архитектура

Никаких новых зависимостей. Данные собираются на бэкенде через `/proc` и системные утилиты, передаются одним JSON-запросом, рендерятся на клиенте.

---

## Бэкенд

### Новый маршрут

```
GET /api/server/stats   (защищён middleware auth)
```

### Источники данных

| Метрика | Источник | Метод |
|---------|----------|-------|
| CPU % | `/proc/stat` | два снимка с паузой 200ms, дельта idle/total |
| RAM | `/proc/meminfo` | MemTotal, MemAvailable |
| Диск | `df -h /` | парсинг строки с `/` |
| Процессы | `ps aux --sort=-%cpu --no-headers` | парсинг stdout |

CPU вычисляется как: `used% = 100 * (1 - deltaIdle / deltaTotal)` между двумя снимками через 200ms.

### Структура ответа

```json
{
  "cpu":  { "used": 12.5 },
  "ram":  { "total": 8192, "used": 3400, "free": 4792 },
  "disk": { "total": "50G", "used": "18G", "free": "32G", "pct": 36 },
  "processes": [
    { "pid": 1234, "user": "deploy", "cpu": 5.2, "mem": 1.8, "cmd": "node server.js" }
  ]
}
```

RAM в мегабайтах. Диск — строки как возвращает `df -h`.

## Фронтенд

### Размещение

Новая вкладка «Сервер» на странице `#page-portal`, добавляется последней:

```
[CLAUDE.md] [Состояние] [История] [Решения] [Сервер]
```

### Макет вкладки

```
[🔄 Обновить]                    Обновлено: 14:32:05

┌─────────────┐ ┌─────────────┐ ┌─────────────────────┐

│  12.5%       │ │  3.4 / 8 GB  │ │  18G / 50G           │

1234    deploy     5.2      1.8     node server.js

...
```

### Прогресс-бары (цветовые пороги)

| Значение | Цвет |
|----------|------|
| < 60%    | зелёный |
| 60–85%   | жёлтый |
| > 85%    | красный |

### Таблица процессов

- Все процессы (без ограничения), с вертикальной прокруткой
- По умолчанию сортировка: CPU% по убыванию
- Кликабельные заголовки: **CPU%**, **MEM%**, **COMMAND**
- Клик на активный столбец — переключает направление (asc/desc)
- Клик на другой столбец — меняет сортировку, направление desc по умолчанию (для COMMAND — asc)
- Индикатор: стрелка ▼/▲ рядом с активным заголовком

### Обновление данных

- Данные загружаются при первом открытии вкладки «Сервер»
- Кнопка «Обновить» — повторный запрос к `/api/server/stats`
- Время последнего обновления отображается рядом с кнопкой
- Нет автополлинга

## Ограничения

- Только Linux (использует `/proc` и `ps`)
- CPU-замер добавляет ~200ms к времени ответа API
- `ps aux` отдаёт всё дерево процессов — на загруженном сервере может быть 100+ строк
